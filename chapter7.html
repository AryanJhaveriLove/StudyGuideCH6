Chapter 7 Study Guide

Understanding Software: A Primer for Managers

This chapter explains how managers should classify software, understand software layers, and connect software choices to lock-in, compatibility, usability, and business value.

What to Know

This chapter is about the basic categories of software, where software runs, who can modify it, and how software layers create dependency across an ecosystem.

Main Theme

Managers do not need to be programmers, but they do need to understand what kind of software they are using and how software choices affect flexibility, compatibility, and lock-in.

Manager Takeaway

Good managers ask four questions about software: what it does, who can change the source code, where it runs, and what standards it uses.

Vocabulary

Software

Definition: Software is a set of instructions that tells hardware what to do.

Why it matters: Managers constantly make decisions about software even when they never write code themselves.

Example: Excel, Windows, Salesforce, and Google Docs are all software, but they serve different roles.

Operating System (OS)

Definition: An operating system is the core software layer that manages hardware resources and supports other software.

Why it matters: The operating system affects which applications can run and how users interact with the device.

Example: Windows, macOS, Linux, and iOS are operating systems.

Database Management System (DBMS)

Definition: A DBMS is software used to store, organize, retrieve, and manage structured data.

Why it matters: Many business applications rely on databases to store and manage critical information.

Example: Oracle Database and MySQL are DBMS examples.

User Application

Definition: A user application is software designed to help end users perform specific tasks.

Why it matters: This is the software employees and customers interact with directly, so it strongly affects productivity and user experience.

Example: Excel, Photoshop, and Google Sheets are user applications.

Closed Source (Proprietary) Software

Definition: Closed source software is software whose source code can be changed only by the company or organization that owns it.

Why it matters: Proprietary software gives the owner more control, but it can increase dependence on the vendor.

Example: Microsoft Excel is proprietary software.

Open Source Software

Definition: Open source software is software whose source code is available for others to inspect, modify, and redistribute.

Why it matters: Open source can increase flexibility and reduce reliance on a single vendor.

Example: Linux is open source software.

Local Software

Definition: Local software runs on the user’s own device and does not require network access to perform its core functions.

Why it matters: Local software can work offline and may feel faster, but it is often harder to update and coordinate across many users.

Example: A desktop version of Excel installed on a laptop is local software.

Hosted Software

Definition: Hosted software runs on a remote server and is accessed through a browser or client rather than running primarily on the user’s device.

Why it matters: Hosted software can simplify collaboration and updates, but it depends on networks and service providers.

Example: Google Sheets is hosted software.

Open Standard

Definition: An open standard allows other firms to build compatible products or complementary software without needing special permission from the owner.

Why it matters: Open standards can help ecosystems grow by making compatibility easier.

Example: A broadly compatible file format or platform interface encourages more complementary products.

Closed Standard

Definition: A closed standard requires permission from the owner before other firms can build compatible products or complementary offerings.

Why it matters: Closed standards give tighter control, but can reduce ecosystem openness and outside participation.

Example: A tightly controlled device ecosystem with limited outside compatibility reflects a closed standard approach.

Konana’s Ecosystem Model

Definition: Konana’s Ecosystem Model shows how computing exists in layered dependencies, from hardware up through operating systems, databases, middleware, and applications.

Why it matters: Because each layer depends on the ones below it, one software choice can influence many later choices and create lock-in across the stack.

Example: Choosing one operating system affects which applications and tools a firm can easily use.

Lock-In

Definition: Lock-in occurs when a user or firm becomes dependent on a product or ecosystem and faces difficulty or cost when trying to switch away.

Why it matters: Software ecosystems can trap users through compatibility, training, files, and complementary products.

Example: A company deeply invested in one software stack may find switching expensive and disruptive.

Graphical User Interface (GUI)

Definition: A graphical user interface is a visual interface that lets users interact with software through icons, menus, windows, buttons, and other on-screen controls instead of only typed commands.

Why it matters: GUIs made computers much easier for non-programmers to use.

Example: Drop-down menus, check boxes, and desktop icons are GUI elements.

Command Line

Definition: The command line is a text-based interface where users type instructions directly to the computer.

Why it matters: Before GUIs became common, many computer tasks required command-line knowledge, which made computing less accessible to average users.

Example: Typing a command into Terminal on a Mac or Command Prompt on Windows uses the command line.

Enterprise Software

Definition: Enterprise software is software designed to support the operations, coordination, or decision-making needs of an organization.

Why it matters: Managers often choose enterprise software to support large-scale business processes rather than only individual tasks.

Example: ERP, CRM, and business intelligence systems are enterprise software categories.

ERP (Enterprise Resource Planning)

Definition: ERP is integrated software that connects multiple business functions, such as accounting, inventory, operations, and HR, into a shared system.

Why it matters: ERP can improve coordination across the business, but it also affects many processes at once.

Example: A company using one system to manage orders, payroll, and inventory is using ERP.

CRM, SCM, and BI Systems

Definition: CRM systems manage customer relationships, SCM systems support supply chain operations, and BI systems analyze data to support managerial decisions.

Why it matters: Managers should understand that different software categories solve different business problems.

Example: Salesforce is a CRM example, while a reporting dashboard for executives is a BI example.

Key Concepts

  • Managers can classify software by asking what it does, who can change the code, where it runs, and what standards it uses.
  • Software exists in layers, and lower-layer choices can shape higher-layer options.
  • Open versus closed approaches affect control, compatibility, and ecosystem growth.
  • Local versus hosted software changes convenience, collaboration, and dependence on networks.
  • GUIs made computing far more accessible to non-programmers.
  • Enterprise software matters because it supports large-scale business processes, not just individual productivity.

3. A company is deciding between a proprietary software product and an open source alternative. Which statement is MOST accurate?
